基本使用

安装

到 Hugo Releases 下载适合你的操作系统的版本。

把 hugo （或者是 Windows 的 hugo.exe） 放到你的 环境变量 PATH 所在的目录，命令行输入命令：hugo version，打印Hugo版本信息则安装成功。

创建工程

$ hugo new site 工程路径

创建的工程目录结构如下：

▸ archetypes/  
▸ content/  
▸ data/  
▸ layouts/  
▸ static/   
▸ themes/    
config.toml

archetypes：文件头模板；

content：md文件所在文件夹；

data：变量文件文件夹；

layouts：页面布局文件文件夹；

static：静态资源，js、css、images等；

themes：主题存放路径；

config：部署配置文件；

创建文件

$ hugo new post/test.md

在 content 文件夹下创建post/test.md文件，通过命令创建的md有自动生成的投标信息，例如：

+++
date = "2015-01-08T08:36:54-07:00"
draft = true
title = "about"
+++

使用主题

在Hugo官网或是Github上下载需要使用的Hugo主题，放到theme文件夹内。该文件夹可以放入多个主题，在运行或部署时通过-t指定具体使用哪个。

运行Hugo

Hugo 包含了内置的高性能 web server 。通过简单的运行 hugo server ， 即可启动运行Hugo程序，这个启动的server可以直接对外提供服务（在访问量不大的情况下）。

$ hugo server -t 主题名称 -D

其中通过-t指定主题，通过-D参数表示草稿态的文件可见（即md文件头中draft=true）。

生成静态文件

运行 hugo 命令会在public文件夹下生成网站静态文件，注意运行时 并不会 删除之前生成的文件。这意味着你必须在运行 hugo 命令之前删除你的 public/ 目录（或者你通过 -d/--destination 指定的目录）。 不然的话，你可能有运行遗留在生成目录的错误文件的风险（比如草稿或者未来的文章）。

一个简单的处理方法是使用不同的目录用于开发和部署环境。

启动一个会生成草稿内容（有助于编辑）的 Server ，指定一个不同的目标目录： dev/ 目录

$ hugo server -wDs ~/Code/hugo/docs -d dev

当内容准备好发布时，使用默认的 public/ 目录：

$ hugo -s ~/Code/hugo/docs

这将防止你不小心发布了还未准备好的内容。

生成到publish中的文件可以放入Nginx或是apache服务器或是托管到GitHub上。

高级使用

Hugo命令

$ hugo help

Usage:
  hugo [flags]
  hugo [command]

常用命令:
  new         新建内容或站点
  server      启动一个高兴web服务器
  config      打印网站配置

标志位:
  -b, --baseURL string             root对应的域名,例如：http://spf13.com/
  -D, --buildDrafts                包含草稿态文件
  -E, --buildExpired               包含过期文件
  -F, --buildFuture                包含发表日期是未来时间单位文件
  -c, --contentDir string          content文件夹的系统路径
  -d, --destination string         输出模板路径
  -s, --source string              相对路径的系统路径
  -t, --theme strings              指定主题

Hugo配置

Hugo 需要在源目录查找一个 config.toml 的配置文件。如果这个文件不存在，将会查找 config.yaml，然后是 config.json 。

这个配置文件是一个整站的配置。它给 Hugo 提供了如何构建站点的方式，比如全局的参数和菜单。

下面是一个 toml 格式的带了一些默认值的配置文件。 在 [params] 下面的值将会构成模板里的 .Site.Params 变量：

baseurl: "http://yoursite.example.com/"
contentdir = "content"
layoutdir = "layouts"
publishdir = "public"
builddrafts = false
baseurl = "http://yoursite.example.com/"
canonifyurls = true

[taxonomies]
  category = "categories"
  tag = "tags"

[params]
  description = "Tesla's Awesome Hugo Site"
  author = "Nikola Tesla"

下面是 Hugo 定义好的变量列表，以及他们的默认值，你可以设置他们：

---
# 原型（文件头）路径

# root的主路径, 例如：http://spf13.com/
baseURL:                    ""

buildDrafts:                false
# 包含发布日期未到的文件
buildFuture:                false
# 所有相对content的相对路径，注意这里只影响相对路径，不影响绝对路径
relativeURLs:               false
canonifyURLs:               false
# 配置文件路径（默认 path/config.yaml|json|toml）
config:                     "config.toml"
contentdir:                 "content"
dataDir:                    "data"
defaultExtension:           "html"
defaultLayout:              "post"
disableLiveReload:          false
# 不构建RSS
disableRSS:                 false
# 不构建站点地图文件
disableSitemap:             false
editor:                     ""
footnoteAnchorPrefix:       ""
footnoteReturnLinkContents: ""
# 谷歌分析追踪ID
googleAnalytics:            ""
languageCode:               ""
layoutdir:                  "layouts"
# 启动日志
log:                        false
# 日志文件路径（如果设置，则自动启动日志）
logFile:                    ""
# "yaml", "toml", "json"
metaDataFormat:             "toml"
newContentEditor:           ""
# 不同步文件的修改时间
noTimes:                    false
paginate:                   10

permalinks:

# 代码高亮样式

# 使用的主题

# 详细日志输出

# verbose logging

# 监控文件变化自动重新生成

工程目录结构

Hugo 使用一个单一的目录用来生成完整的站点。

源目录的顶层一般有以下一些元素：

▸ archetypes/
▸ content/
▸ data/
▸ layouts/
▸ static/
▸ themes/
  config.toml

archetypes：文件头模板；

content：md文件所在文件夹；

data：变量文件文件夹；

layouts：页面布局文件文件夹；

static：静态资源，js、css、images等；

themes：主题存放路径；

config：部署配置文件；

一个目录的例子：

.
├── config.toml
├── archetypes
|   └── default.md
├── content
|   ├── post
|   |   ├── firstpost.md
|   |   └── secondpost.md
|   └── quote
|   |   ├── first.md
|   |   └── second.md
├── data
├── layouts
|   ├── _default
|   |   ├── single.html
|   |   └── list.html
|   ├── partials
|   |   ├── header.html
|   |   └── footer.html
|   ├── taxonomies
|   |   ├── category.html
|   |   ├── post.html
|   |   ├── quote.html
|   |   └── tag.html
|   ├── post
|   |   ├── li.html
|   |   ├── single.html
|   |   └── summary.html
|   ├── quote
|   |   ├── li.html
|   |   ├── single.html
|   |   └── summary.html
|   ├── shortcodes
|   |   ├── img.html
|   |   ├── vimeo.html
|   |   └── youtube.html
|   ├── index.html
|   └── sitemap.xml
├── themes
|   ├── hyde
|   └── doc
└── static
    ├── css
    └── js

这个目录结构告诉了我们一些关于这个站点的信息：

  1. 这个网站打算有两个不同类型的文章内容：posts 和 quotes 。
  2. 它将在文章内容里应用两个不同的分类：categories 和 tags 。
  3. 它将在三种不同的视图里呈现这些内容：列表、摘要和完整的内容。

data

除了内置的变量，Hugo还支持自定义变量，这些变量可以在data文件夹下的yaml、json、toml文件中定义。定义的变量也可以用于主题的构建。这些变量呗统一的放入 .Site.Data这个map中，map中的key是根据文件路径.文件名.变量名。

archetypes

Hugo引入了内容生成器的概念，通过hugo new命令可以生成文件，并且在文件头部自动携带title、date等信息，如果默认携带的信息不满足要求，可以通过自定义原型文件（archetypes）的方式定义 额外的 头部信息，这样在执行hugo new时即可携带上自定义的头部信息。

默认的信息可以放在archetype/default.md中，对于不同文件需要不同的头部信息的情况，可以通过在 archetypes中定义文件名称与content下子目录名称一致的方式，为不同的文件夹定制不同的文件头，例如 content/musician对应 archetypes下的 musician.md 。

多个archetype文件的生效优先级：

  • 根目录archetypes下文件名称与路径匹配的；
  • 根目录archetypes下的/default.md；
  • 主题archetypes下文件名称与路径匹配的；
  • 主题archetypes下的/default.md；
  • Hugo默认的（里面只有title、date）；

支持格式包括：

TOML：“+++”；

YAML：“—”

JSON：“{}”

在文件头中定义的变量可以在模板中通过。Params.key获取，其中key是全小写变量名称。

常见变量：

  • title：文章标题；
  • description： 文章描述信息
  • date：当前时间
  • taxonomies：分类信息，例如categories、tags；

可选变量：

  • aliases：更新路径后的旧路径列表，会生成旧路径到当前文件的链接；
  • draft：是否草稿；
  • publishdate：发布日期，未来日期不发布；
  • type：文章类型，与上面的type对应；
  • isCJKLanguage：是否是中日韩语言；
  • weight：排序权重，优先级高于date；
  • slug：路径后缀指定；
  • url：全路径指定；

content

Hugo支持content下文件的任意层嵌套，但是最上层文件夹是特殊的，被称作section。md文件与生成文件的对于关系如下 ：

.
└── content
    ├── post
    |   ├── firstpost.md   // <- http://1.com/post/firstpost/index.html
    |   ├── happy
    |   |   └── ness.md    // <- http://1.com/post/happy/ness/index.html
    |   └── secondpost.md  // <- http://1.com/post/secondpost/index.html
    └── quote
        ├── first.md       // <- http://1.com/quote/first/index.html
        └── second.md      // <- http://1.com/quote/second/index.html

如果启动或生成时设置了hugo --uglyURLs，那么生成的对于关系如下：

.
└── content
    ├── post
    |   ├── firstpost.md   // <- http://1.com/post/firstpost.html
    |   ├── happy
    |   |   └── ness.md    // <- http://1.com/post/happy/ness.html
    |   └── secondpost.md  // <- http://1.com/post/secondpost.html
    └── quote
        ├── first.md       // <- http://1.com/quote/first.html
        └── second.md      // <- http://1.com/quote/second.html

对于某些文件不想通过默认的方式生成路径的情况，Hugo允许在文件头中定义额外属性达到单独控制文件输出路径的目的，一下几种配置按照优先级从低到高的顺序起作用：

filename：不在文件头中，但是默认情况下文件名决定了文件的路径；

slug：定义在文件头中，可以覆盖filename，决定文件路径；

filepath：生成文件按照md文件的路径（从section那一层开始）生成路径；

section：及content下的第一层，不可被任何文件头属性修改；

type：可在文件头中定义，type对应这一套完整的文件，包括：layouts下的typename文件夹，该文件夹下的single.html、list.html，archetypes下的typename.md，他们会和content下typename文件夹下的内容对应生效；

path：可在文件头中定义，会覆盖文件的真实路径；

url：可以覆盖上面的所有配置，必须是一个从根路径开始的路径；

文章概要

可以通过在文件头中指定Summary变量来设置文章在列表页中的概要内容，同时也可以在文章中通过 <!–more–> 标签，截取标签前的内容作为文章概要。 <!–more–> 标签的优先级高于手动设置Summary变量。

主题

修改主题

主题中的静态文件（static文件夹下的内容）、模板文件（layouts文件夹下的内容）、文件头原型文件（archetypes文件夹下的内容）都可以通过在工程工作空间中的对应文件夹的内容进行覆盖。覆盖时注意相对路径保持对应。Hugo在加载主题时会先加载本地工作空间中的内容，然后再去主题文件夹中加载。

创建主题

Hugo可以通过命令hugo new theme [name]创建主题，该命令会初始化主题所需的所有文件，在其基础上进行主题的定制，创建的主题包含内容如下：

.
├── archetypes
|   └── default.md
├── layouts
|   ├── _default
|   |   ├── baseof.html
|   |   └── list.html
|   |   └── single.html
|   └── partials
|   |   ├── footer.html
|   |   └── head.html
|   |   └── header.html
|   ├── 404.html
|   ├── index.html
├── static
|   ├── css
|   ├── js
├── theme.toml

archetypes：定义在文件头中使用的特殊的key，如有需要可以为每个content type创建对应的archetype文件；

layouts：提供默认布局模板，可以在其中按照content type或是section自定义模板；

  • 单体模板：single.html，文件内容展示模板；
  • 列表模板：list.html文章列表页的模板；
  • 局部模板：页面局部模板，可以被其他模板引用，方便最小修改、代码复用；

static：主要存放js、css、图片等静态文件，部署时会被直接拷贝到目标工程；

模板

变量

page变量：文件头中的保留变量直接通过.name获取，其余的自定义的可以通过.Params.key获取；

node变量：分类、list、homepage模板中可以使用的变量，一些保留名称的直接通过.name获取，定义在data文件夹中的可以通过.Data.key获取；

site变量：全局变量，通过.Site.key获取；

单体模板

模板生效顺序：

  • /layouts/TYPE-or-SECTION/LAYOUT.html
  • /layouts/TYPE-or-SECTION/single.html
  • /layouts/_default/single.html
  • /themes/THEME/layouts/TYPE-or-SECTION/LAYOUT.html
  • /themes/THEME/layouts/TYPE-or-SECTION/single.html
  • /themes/THEME/layouts/_default/single.html

模板样例：

{{ partial "header.html" . }}
{{ partial "subheader.html" . }}
{{ $baseurl := .Site.BaseURL }}

<section id="main">
  <h1 id="title">{{ .Title }}</h1>
  <div>
        <article id="content">
           {{ .Content }}
        </article>
  </div>
</section>

<aside id="meta">
    <div>
    <section>
      <h4 id="date"> {{ .Date.Format "Mon Jan 2, 2006" }} </h4>
      <h5 id="wc"> {{ .FuzzyWordCount }} Words </h5>
    </section>
    <ul id="categories">
      {{ range .Params.topics }}
        <li><a href="{{ $baseurl }}/topics/{{ . | urlize }}">{{ . }}</a> </li>
      {{ end }}
    </ul>
    <ul id="tags">
      {{ range .Params.tags }}
        <li> <a href="{{ $baseurl }}/tags/{{ . | urlize }}">{{ . }}</a> </li>
      {{ end }}
    </ul>
    </div>
    <div>
        {{ if .Prev }}
          <a class="previous" href="{{.Prev.Permalink}}"> {{.Prev.Title}}</a>
        {{ end }}
        {{ if .Next }}
          <a class="next" href="{{.Next.Permalink}}"> {{.Next.Title}}</a>
        {{ end }}
    </div>
</aside>

{{ partial "disqus.html" . }}
{{ partial "footer.html" . }}

列表模板

section列表模板生效顺序：

  • /layouts/section/SECTION.html
  • /layouts/_default/section.html
  • /layouts/_default/list.html
  • /themes/THEME/layouts/section/SECTION.html
  • /themes/THEME/layouts/_default/section.html
  • /themes/THEME/layouts/_default/list.html

分类列表模板生效顺序：

  • /layouts/taxonomy/SINGULAR.html (e.g. /layouts/taxonomy/topic.html)
  • /layouts/_default/taxonomy.html
  • /layouts/_default/list.html
  • /themes/THEME/layouts/taxonomy/SINGULAR.html
  • /themes/THEME/layouts/_default/taxonomy.html
  • /themes/THEME/layouts/_default/list.html

模板样例：

{{ partial "header.html" . }}
{{ partial "subheader.html" . }}

<section id="main">
  <div>
   <h1 id="title">{{ .Title }}</h1>
    {{ range .Data.Pages }}
        {{ .Render "summary"}}
    {{ end }}
  </div>
</section>

{{ partial "footer.html" . }}

主页模板

模板生效顺序：

  • /layouts/index.html
  • /layouts/_default/list.html
  • /layouts/_default/single.html
  • /themes/THEME/layouts/index.html
  • /themes/THEME/layouts/_default/list.html
  • /themes/THEME/layouts/_default/single.html

模板样例：

<!DOCTYPE html>
<html class="no-js" lang="en-US" prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb#">
<head>
    <meta charset="utf-8">

    {{ partial "meta.html" . }}

    <base href="{{ .Site.BaseURL }}">
    <title>{{ .Site.Title }}</title>
    <link rel="canonical" href="{{ .Permalink }}">
    <link href="{{ .RSSlink }}" rel="alternate" type="application/rss+xml" title="{{ .Site.Title }}" />

    {{ partial "head_includes.html" . }}
</head>
<body lang="en">

{{ partial "subheader.html" . }}

<section id="main">
  <div>
    {{ range first 10 .Data.Pages }}
        {{ .Render "summary"}}
    {{ end }}
  </div>
</section>

{{ partial "footer.html" . }}

分组展示模板

模板生效顺序：

  • /layouts/taxonomy/SINGULAR.terms.html (e.g. /layouts/taxonomy/topic.terms.html)
  • /layouts/_default/terms.html

模板特殊变量：

  • .Data.Singular：分组的单数名称；
  • .Data.Plural：分组的复数名称；
  • .Data.Terms：分组；
  • .Data.Terms.Alphabetical：按字母排序；
  • .Data.Terms.ByCount：按流行程度排序；

模板样例：

{{ partial "header.html" . }}
{{ partial "subheader.html" . }}

<section id="main">
  <div>
    <h1 id="title">{{ .Title }}</h1>

    <ul>
    {{ $data := .Data }}
    {{ range $key, $value := .Data.Terms }}
      <li><a href="{{ $data.Plural }}/{{ $key | urlize }}">{{ $key }}</a> {{ len $value }}</li>
    {{ end }}
   </ul>
  </div>
</section>

{{ partial "footer.html" . }}

内容视图

可复用的、可定制的、可抽取出来的页面中的一部分内容的显示模板。

模板生效顺序：

  • /layouts/TYPE/VIEW.html
  • /layouts/_default/VIEW.html
  • /themes/THEME/layouts/TYPE/VIEW.html
  • /themes/THEME/layouts/_default/view.html

模板样例（在list中使用）：

<section id="main">
<div>
<h1 id="title">{{ .Title }}</h1>
{{ range .Data.Pages }}
{{ .Render "summary"}}
{{ end }}
</div>
</section>

<article class="post">
<header>
<h2><a href='{{ .Permalink }}'> {{ .Title }}</a> </h2>
<div class="post-meta">{{ .Date.Format "Mon, Jan 2, 2006" }} - {{ .FuzzyWordCount }} Words </div>
</header>

{{ .Summary }}
<footer>
<a href='{{ .Permalink }}'><nobr>Read more →</nobr></a>
</footer>
</article>

其中{{ .Render "summary"}}即为使用content view的代码。
